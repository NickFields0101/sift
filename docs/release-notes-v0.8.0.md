# SIFT v0.8.0

Version 0.8.0 adds SIFT's first production Python intelligence engine while
keeping the deterministic TypeScript decision system authoritative.

## New intelligence workflow

- Generate & Screen now runs a bounded competitor map and red-team pass after
  optional public research.
- The result identifies alternatives, fatal assumptions, counterarguments,
  disconfirming tests, and explicit go-forward conditions.
- Live progress appears inside the existing one-click workflow.
- The complete provisional analysis is available from the thesis result.
- Local Ollama, LM Studio, OpenRouter, and approved OpenAI-compatible models can
  run the analysis. Only OpenRouter supplies attributable public web retrieval.

## Trust and evidence boundaries

- Python proposes; TypeScript validates and applies.
- The Python process cannot write project state, calculate SIFT's score, resolve
  gates, verify evidence, run build tools, or approve a decision.
- Every result is required to state that it is provisional public context and
  not customer validation.
- Supplied source excerpts retain their exact source IDs and SHA-256 integrity
  boundary. Unsupported competitor statements remain model hypotheses.
- Interviews, commitments, payments, tests, production behavior, and audits
  still begin only after a new idea enters validation.

## Desktop packaging and reliability

- Windows and macOS packages embed a native Python worker; users do not install
  Python or Docker.
- Electron supervises the worker over private JSONL standard input/output with
  strict schemas, bounded budgets, live polling, cancellation, timeouts, and
  crash cleanup.
- API keys remain in operating-system-protected SIFT settings and are passed to
  the worker only inside the private request pipe. They never appear in process
  arguments, environment variables, progress events, or renderer output.
- If the worker is missing, incompatible, or fails, Generate & Screen continues
  through the established TypeScript path rather than treating the idea as
  failed validation.

## Compatibility

- Installing SIFT 0.8.0 over 0.7.0 preserves the existing application identity,
  local projects, and protected model settings.
- Deterministic scoring and evidence policies are unchanged.
- The hosted web edition remains functional and simply omits the desktop-only
  Python pass.
